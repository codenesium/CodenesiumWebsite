Controllers

Controllers should be thin and have no business logic.

If you live by this rule your life will be easier.

In our REST API they take the request and pass it to the service layer.

On the response side they return a response with a status code for successes and failure.

Imagine you don’t want to use your service in a web app and instead want to use it in a service bus. The goal is to keep all business logic in the service layer.
